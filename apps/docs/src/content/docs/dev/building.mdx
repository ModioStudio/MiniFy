# Building from Source

This guide walks you through building MiniFy from source code.

## Prerequisites

Before building, ensure you have the following installed:

### All Platforms

- [Node.js](https://nodejs.org/) 18 or later
- [pnpm](https://pnpm.io/) 9 or later
- [Rust](https://rustup.rs/) (latest stable)
- [Git](https://git-scm.com/)

### Platform-Specific Requirements

<Tabs>
  <TabItem label="Windows">
    - Microsoft Visual Studio C++ Build Tools
    - WebView2 (usually pre-installed on Windows 10/11)

    ```powershell
    # Install Visual Studio Build Tools
    winget install Microsoft.VisualStudio.2022.BuildTools
    ```
  </TabItem>
  <TabItem label="macOS">
    - Xcode Command Line Tools

    ```bash
    xcode-select --install
    ```
  </TabItem>
  <TabItem label="Linux">
    - Various development libraries

    ```bash

      build-essential \

      wget \
      file \
      libxdo-dev \
      libssl-dev \
      libayatana-appindicator3-dev \
      librsvg2-dev

      openssl-devel \
      curl \
      wget \
      file \
      libxdo-devel \
      libappindicator-gtk3-devel \
      librsvg2-devel
    ```
  </TabItem>
</Tabs>

## Clone the Repository

```bash
git clone https://github.com/ModioStudio/MiniFy.git
cd MiniFy
```

## Install Dependencies

```bash
pnpm install
```

This installs all dependencies for the monorepo, including:
- Desktop app (React + Tauri)
- Website (Next.js)
- Documentation (Astro + Starlight)

## Development Build

Run the desktop app in development mode:

```bash
pnpm desktop:dev
```

This will:
1. Start the Vite development server
2. Compile the Rust backend
3. Open the development window

<Aside type="tip">
  Hot reload is enabled for the React frontend. Changes to Rust code require a rebuild.
</Aside>

## Production Build

<Steps>
1. **Build the application**
   
   ```bash
   pnpm desktop:build
   ```

2. **Find the output**
   
   Built installers are located in:
   ```
   apps/desktop/src-tauri/target/release/bundle/
   ```

3. **Available formats**
   
   - **Windows**: `.msi`, `.exe`
   - **macOS**: `.dmg`, `.app`
   - **Linux**: `.deb`, `.rpm`, `.AppImage`
</Steps>

## Build Scripts

| Script | Description |
|--------|-------------|
| `pnpm desktop:dev` | Run desktop app in development |
| `pnpm desktop:build` | Build desktop app for production |
| `pnpm www:dev` | Run website in development |
| `pnpm www:build` | Build website for production |
| `pnpm docs:dev` | Run documentation in development |
| `pnpm docs:build` | Build documentation for production |
| `pnpm build` | Build all packages |
| `pnpm dev` | Run all packages in development |

## Environment Variables

Create a `.env` file in `apps/desktop/` if needed:

```bash
# Spotify OAuth credentials (optional for development)
VITE_SPOTIFY_CLIENT_ID=your_client_id
```

<Aside type="note">
  The default client ID works for local development. You only need custom credentials for production builds.
</Aside>

## Troubleshooting

### Rust compilation fails

```bash
# Update Rust
rustup update

# Clear Cargo cache
cargo clean
```

### Node modules issues

```bash
# Clear node_modules and reinstall
rm -rf node_modules
rm -rf apps/*/node_modules
pnpm install
```

### Tauri CLI issues

```bash
# Reinstall Tauri CLI
pnpm add -D @tauri-apps/cli@latest
```

### WebView2 issues (Windows)

Download and install WebView2 from [Microsoft](https://developer.microsoft.com/en-us/microsoft-edge/webview2/).

### GTK issues (Linux)

Ensure all GTK dependencies are installed:

```bash
sudo apt install libgtk-3-dev libwebkit2gtk-4.1-dev
```

## Building for Multiple Platforms

Tauri supports cross-compilation in some cases, but we recommend building on each target platform for best results.

### GitHub Actions

The repository includes CI/CD workflows that build for all platforms:

```yaml
# .github/workflows/build.yml
on:
  push:
    tags:
      - 'v*'
```
