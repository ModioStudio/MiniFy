# Custom Themes

MiniFy supports fully customizable themes. This guide walks you through creating your own theme
from scratch, understanding the theme system, and sharing themes with the community.

## Theme File Locations

### Built-in Themes

Built-in themes are bundled with the application:

```

├── dark.json
├── light.json
├── catppuccin.json
├── dracula.json
├── chatgpt.json
├── youtube.json
├── bmw.json
```

### Custom Theme Locations

Custom themes should be placed in the user themes directory:

<Tabs>
  <TabItem label="Windows">
    ```
    %APPDATA%\MiniFy\themes\
    ```
    Example: `C:\Users\YourName\AppData\Roaming\MiniFy\themes\`
  </TabItem>
  <TabItem label="macOS">
    ```
    ~/Library/Application Support/MiniFy/themes/
    ```
  </TabItem>
  <TabItem label="Linux">
    ```
    ~/.config/MiniFy/themes/
    ```
  </TabItem>
</Tabs>

## Creating a Custom Theme

<Steps>
1. **Create the themes directory**

   <Tabs>
     <TabItem label="Windows">
       ```powershell
       New-Item -ItemType Directory -Force "$env:APPDATA\MiniFy\themes"
       ```
     </TabItem>
     <TabItem label="macOS">
       ```bash
       mkdir -p ~/Library/Application\ Support/MiniFy/themes
       ```
     </TabItem>
     <TabItem label="Linux">
       ```bash
       mkdir -p ~/.config/MiniFy/themes
       ```
     </TabItem>
   </Tabs>

2. **Create your theme file**

   Create a new JSON file (e.g., `my-theme.json`):

   ```json
   {
     "name": "My Custom Theme",
     "author": "Your Name",
     "version": "1.0.0",
     "description": "A brief description of your theme",
     "colors": {
       "background": "#1e1e2e",
       "foreground": "#cdd6f4",
       "primary": "#89b4fa",
       "secondary": "#313244",
       "accent": "#f5c2e7",
       "muted": "#6c7086",
       "border": "#45475a"
     }
   }
   ```

3. **Restart MiniFy**

   Close and reopen MiniFy to load the new theme.

4. **Select your theme**

   Right-click → Themes → Select "My Custom Theme"
</Steps>

## Theme Schema Reference

### Complete Schema

```json
{
  "name": "Theme Name",
  "author": "Author Name",
  "version": "1.0.0",
  "description": "Theme description",
  "colors": {
    "background": "#0a0a0a",
    "foreground": "#ffffff",
    "primary": "#1DB954",
    "secondary": "#1a1a1a",
    "accent": "#1ed760",
    "muted": "#666666",
    "border": "#2a2a2a",
    "success": "#22c55e",
    "warning": "#f59e0b",
    "error": "#ef4444"
  }
}
```

### Required Fields

| Field | Type | Description |
|-------|------|-------------|
| `name` | string | Display name in theme menu |
| `colors` | object | Color definitions |

### Optional Fields

| Field | Type | Description |
|-------|------|-------------|
| `author` | string | Theme creator's name |
| `version` | string | Theme version (semver) |
| `description` | string | Brief description |

### Color Properties

| Property | CSS Variable | Usage |
|----------|--------------|-------|
| `background` | `--background` | Main window background |
| `foreground` | `--foreground` | Primary text color |
| `primary` | `--primary` | Accent color (buttons, progress) |
| `secondary` | `--secondary` | Secondary backgrounds (cards) |
| `accent` | `--accent` | Hover states, highlights |
| `muted` | `--muted` | Subdued text (timestamps) |
| `border` | `--border` | Border colors, dividers |
| `success` | `--success` | Success states (optional) |
| `warning` | `--warning` | Warning states (optional) |
| `error` | `--error` | Error states (optional) |

## Color Formats

Colors can be specified in multiple formats:

```json
{
  "colors": {
    "background": "#0a0a0a",
    "foreground": "rgb(255, 255, 255)",
    "primary": "hsl(142, 76%, 36%)",
    "accent": "rgba(30, 215, 96, 0.8)"
  }
}
```

<Aside type="tip">
**Hex colors (#RRGGBB)** are recommended for consistency and ease of editing.
</Aside>

## Color Design Tips

### Contrast Guidelines

```
┌───────────────────────────────────────────────────────────────────────────────┐
│                        Contrast Recommendations                                │
├───────────────────────────────────────────────────────────────────────────────┤
│                                                                                │
│   Background vs Foreground:                                                   │
│   ─────────────────────────                                                   │
│   • Minimum contrast ratio: 4.5:1 (WCAG AA)                                   │
│   • Recommended: 7:1 or higher (WCAG AAA)                                     │
│                                                                                │
│   Primary vs Background:                                                       │
│   ──────────────────────                                                      │
│   • Should be visually distinct                                               │
│   • Used for interactive elements                                              │
│                                                                                │
│   Muted vs Background:                                                         │
│   ────────────────────                                                        │
│   • Should be readable but subdued                                            │
│   • Minimum contrast ratio: 3:1                                               │
│                                                                                │
└────────────────────────────────────────────────────────────────────────────────┘
```

### Color Palette Tools

- [Coolors](https://coolors.co) - Color scheme generator
- [ColorHunt](https://colorhunt.co) - Curated color palettes
- [Contrast Checker](https://webaim.org/resources/contrastchecker/) - WCAG contrast tool
- [Paletton](https://paletton.com) - Color wheel and harmonies

### Theme Testing Checklist

Before sharing your theme, test these scenarios:

- [ ] Track title is readable
- [ ] Artist name is readable but subdued
- [ ] Progress bar is clearly visible
- [ ] Buttons have clear hover states
- [ ] Active/inactive states are distinguishable
- [ ] Works in all three layouts
- [ ] Settings page is usable
- [ ] AI DJ chat is readable

## Managing Themes

### In-App Theme Management

1. Right-click → Settings → Themes
2. View all themes (built-in + custom)
3. Delete custom themes
4. Export themes for sharing

### Theme Validation

MiniFy validates themes on load. Invalid themes are skipped with a console warning.

**Common validation errors:**
- Missing `name` field
- Missing `colors` object
- Invalid color format
- JSON syntax errors

### Debugging Theme Issues

Open DevTools (F12) to see theme loading errors:

```javascript
// Console output
[Theme] Loading theme: my-theme.json
[Theme] Error: Invalid color format for 'primary'
```

## Sharing Themes

### Contributing to MiniFy

<Steps>
1. **Fork the repository**

   [Fork on GitHub](https://github.com/ModioStudio/MiniFy)

2. **Add your theme**

   Add the JSON file to `apps/desktop/src/themes/`

3. **Add to website preview**

   Also add to `apps/www/public/themes/`

4. **Update theme index**

   Add your theme to `apps/www/public/themes/index.json`

5. **Submit a Pull Request**

   Include screenshots showing your theme in use
</Steps>

### Sharing Externally

You can share theme JSON files directly:

1. Export your theme from Settings
2. Share the JSON file
3. Recipients place it in their themes directory

## Advanced Customization

### CSS Variable Reference

All theme colors are available as CSS custom properties:

```css
:root {
  --background: #0a0a0a;
  --foreground: #ffffff;
  --primary: #1DB954;
  --secondary: #1a1a1a;
  --accent: #1ed760;
  --muted: #666666;
  --border: #2a2a2a;
  --success: #22c55e;
  --warning: #f59e0b;
  --error: #ef4444;
}
```

### Derived Colors

The theme system also generates derived colors:

```css
:root {
  /* Settings panel specific */
  --settings-panel-bg: /* derived from secondary */;
  --settings-panel-border: /* derived from border */;
  --settings-text: /* derived from foreground */;
  --settings-text-muted: /* derived from muted */;
  --settings-accent: /* derived from primary */;
}
```

## Troubleshooting

### Theme not appearing in menu

1. Check the file is valid JSON
2. Verify it's in the correct directory
3. Ensure the file has a `.json` extension
4. Restart MiniFy

### Colors not applying correctly

1. Check for typos in color property names
2. Verify color format is valid
3. Check DevTools console for errors

### Theme causes rendering issues

1. Ensure adequate contrast between colors
2. Test all UI elements
3. Try adjusting problematic colors
